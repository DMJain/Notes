### Quick Comparison

| Aspect | Process | Thread |
|--------|---------|--------|
| **Memory** | Isolated (own heap) | Shared (same heap) |
| **Creation Cost** | Heavy (~100ms) | Light (~1ms) |
| **Communication** | IPC (pipes, sockets) | Direct memory access |
| **Context Switch** | Expensive | Cheaper |
| **Crash Impact** | Only that process | May crash entire process |

### Thread States Explained

| State | Description | How to Enter |
|-------|-------------|--------------|
| **NEW** | Thread created but not started | `new Thread()` |
| **RUNNABLE** | Ready to run or currently running | `t.start()` |
| **BLOCKED** | Waiting for monitor lock | Trying to enter `synchronized` block held by another |
| **WAITING** | Waiting indefinitely for another thread | `wait()`, `join()` |
| **TIMED_WAITING** | Waiting for specified time | `sleep(ms)`, `join(ms)`, `wait(ms)` |
| **TERMINATED** | Thread completed execution | `run()` completes or exception |

### 3. `interrupt()`

```java
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // Keep working until interrupted
        doWork();
    }
    System.out.println("Thread interrupted, cleaning up...");
});

t.start();
Thread.sleep(2000);
t.interrupt();  // Signal thread to stop
```

## When to Use Threads

### ‚úÖ Good Use Cases

| Scenario | Why Threads Help |
|----------|-----------------|
| **I/O-bound tasks** | Don't block while waiting for file/network |

| **Parallel computation** | Use all CPU cores for heavy math |
| **Timers/Schedulers** | Periodic tasks without freezing main |
| **Event handling** | Respond to events asynchronously |

## When NOT to Use Threads

### ‚ùå Anti-Patterns

| Scenario | Why Threads are WRONG |
|----------|----------------------|
| **Simple sequential logic** | Threads add complexity with no benefit |
| **Shared mutable state without sync** | Race conditions, data corruption |
| **Very short tasks** | Thread overhead > task time |
| **Dependent operations** | Step 2 needs Step 1's result |

### Example: Overhead Trap

```java
// ‚ùå BAD: Thread for tiny task
for (int i = 0; i < 1000; i++) {
    new Thread(() -> {
        System.out.println(i);  // 0.001ms task
    }).start();                  // 1ms thread overhead!
}
// Thread creation is 1000x more expensive than the task!

// ‚úÖ GOOD: Batch work or use thread pools
ExecutorService pool = Executors.newFixedThreadPool(4);
for (int i = 0; i < 1000; i++) {
    pool.submit(() -> System.out.println(i));
}
pool.shutdown();
```

## Common Gotchas

### 1. Calling `run()` Instead of `start()`

```java
Thread t = new Thread(() -> System.out.println("New thread: " + 
                            Thread.currentThread().getName()));

t.run();   // ‚ùå Prints: "New thread: main" (wrong thread!)
t.start(); // ‚úÖ Prints: "New thread: Thread-0" (correct!)
```

### 2. Calling `start()` Twice

```java
Thread t = new Thread(() -> System.out.println("Hello"));
t.start();
t.start();  // ‚ùå IllegalThreadStateException!

// A thread can only be started ONCE
```

### 3. Ignoring InterruptedException

```java
// ‚ùå BAD: Swallowing the exception
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    // Ignored - thread won't respond to interrupts!
}

// ‚úÖ GOOD: Restore interrupt status
try {
    Thread.sleep(1000);
} catch (InterruptedException e) {
    Thread.currentThread().interrupt();  // Preserve the interrupt
}
```

### 4. Thread Safety (Preview for Next Chapter)

```java
// ‚ùå DANGER: Shared mutable state
int counter = 0;

Thread t1 = new Thread(() -> { for(int i=0; i<1000; i++) counter++; });
Thread t2 = new Thread(() -> { for(int i=0; i<1000; i++) counter++; });

t1.start(); t2.start();
t1.join();  t2.join();

System.out.println(counter);  // Expected: 2000, Actual: 1500-2000 (varies!)

// This is a RACE CONDITION - we'll fix this in the synchronization chapter
```

## Summary

| Concept | Key Point |
|---------|-----------|
| **Thread** | Smallest unit of execution within a process |
| **Runnable** | Interface for defining tasks (preferred over extending Thread) |
| **start()** | Creates new thread and calls run() on it |
| **run()** | The code that executes on the thread |
| **join()** | Wait for thread to complete |
| **sleep()** | Pause thread for specified time |
| **Thread States** | NEW ‚Üí RUNNABLE ‚Üí WAITING/BLOCKED ‚Üí TERMINATED |
