### Common Scheduling Algorithms

| Algorithm | How It Works | Best For |
|-----------|-------------|----------|
| **FCFS** (First-Come-First-Serve) | Execute in order of arrival | Simple batch jobs |
| **Round Robin** | Each process gets fixed time slice | Interactive systems |
| **Priority** | Higher priority runs first | Real-time systems |
| **Shortest Job Next** | Shortest task runs first | Minimizing wait time |

## When NOT to Use Concurrency

### âŒ Anti-Patterns

| Scenario | Why Concurrency is WRONG |
|----------|-------------------------|
| **Simple sequential logic** | Adding threads = added complexity for no benefit |
| **Tightly coupled operations** | Step 2 needs Step 1's resultâ€”can't parallelize |
| **Single-threaded libraries** | Some libraries aren't thread-safe (e.g., older GUI toolkits) |
| **Very small tasks** | Thread creation overhead > time saved |
| **Shared state without sync** | Race conditions, data corruption |

### Example: When NOT to Use Threads

```java
// âŒ BAD: Sequential logic forced into threads
public void calculateTax() {
    Thread t1 = new Thread(() -> fetchIncome());     // Step 1
    Thread t2 = new Thread(() -> applyDeductions()); // Step 2 (needs Step 1!)
    Thread t3 = new Thread(() -> computeTax());      // Step 3 (needs Step 2!)
    
    // These CANNOT run in parallelâ€”each depends on the previous!
    // Threads add overhead with ZERO benefit
}

// âœ… GOOD: Just run sequentially
public void calculateTax() {
    int income = fetchIncome();
    int deductions = applyDeductions(income);
    int tax = computeTax(income, deductions);
}
```
